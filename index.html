Project idea:

Overview

The project will be a web and mobile based application that will compare the prices of grocery items, as compared to other stores, which will involve the use of a web scraper to collect all the links available. It will then scrape the data, aggregate, and clean the data. This cleaned data will then be used to identify what the item is with the use of the category and name, the price of the product and whether it is on sale or not, compare the price of similar products for stores around you, and show stock availability, depending on store, and then store data in some sort of database.

Motivation

Everyone needs to buy things from the grocery store, but it is time consuming to find the best prices from every grocery store competitor. If a consumer could be given the ability to search across multiple grocery stores to find the best prices and given a distance to the store, based off distance travel time data given from Google Directions API, this information would allow them to make an informed decision and evaluate what is best for them, given possible advantages due to store times, rewards, distance, and price. Although the processes of crawling, scraping, cleaning, and storing the data to be processed will take some tweaking to automate, it is feasibly possible.

Description

As for features for the application, a search bar will be required to be able to search through the catalogue of items that are stored in the database. The user is aided by the ability to filter through items based on store, brands, price (unit or by item), and range from store in KMs, such that it will only show stores within a certain distance of the user. At this point you can either choose a store in which you wish to shop, or in the case the user has selected the setting in their account which enables “replacement if available”, which will try to find a related item based off the name, category or type of item, and price. Upon searching for an item, if you click on that item, you will be brought to a page that will show the current price, product information such as nutritional information and weight, store(s) of which stock that specific product, product reviews scraped from either the website or alternatively product review or something similar, and historical price of the item, which would allow the user to compare the price to previous prices, to ensure they’re getting the best price.

The application will use a graphical user interface to show the user the surrounding stores, depending on limiting factors, such as the store being open or not. This graphic user interface will show the total price of the shopping list on the pins, an icon of the brand of store, and depending on whether the user agrees to replacement items for different stores or not will be a limiting factor for the number of stores that show up, relative to the position of the user.

Through the use of a personal account, the application will be able to keep track of who and where the user is, enabling the user to get personal notifications for product price drops via email, and get positional data through a stored home address or current location to ensure that the distance measurements from stores are correct. This allows a user to understand the distances to stores and have that be part of the decision.

Once the user has decided which store they wish to go to, the user can click on the corresponding store pin, depending on the store, as not all stores show aisle information for products, in the case the store does give the aisle information for products, such as either Woolworths or Coles, the application will use that information to display aisle information.

This aisle information will allow the user to understand in which order of aisles to go to for the store if they choose to use the application to organise their shopping. It will also display other store information, such as the store address, opening and closing hours, and the phone number. The user can click onto the address, which will be used to link to their choice of map application, most likely to be Google maps, and use that as a guide to the store.

Tools and Technologies

The website itself would most likely be a cloud-based server, as it is cheap and easy to deploy, most likely a version of linux, such as Apache HTTP Server, of which can be hosted by many sources, such as HostGator Web Hosting. An application server will be required to run the web application, which will require most commonly uses an operating system, such as Apache Tomcat.

And the application server will most likely go along with a database server, being most likely a linux based operating system, of which is most commonly MySQL. MySQL is an open-source relational database management system that can be hosted by cloud servers such as Microsoft Azure, Amazon Elastic Compute Cloud, and Oracle Cloud Infrastructure; alternatively, could be hosted by yourself, which could be cheaper.

Skills Required

For large scale deployment, it would require servers, in which would require proficiency in whatever server architecture used, be it the operating system, tools or physical hardware, in the case you’re not using cloud servers. It could be required to use Python for the web crawling, data scraping, and storage of the data. Depending on the type of database management system you use, it will be likely you will need to understand PHP & SQL/MySQL for the database and the search engine required. HTTP/CSS will be required for the website, along with supporting languages, such as Java, the ability to use APIs and their associated languages, namely JavaScript.

Outcome

If the project is successful, the user will be able to utilise the web and or the mobile application to make an informed decision of where they want to go to get what items, based off price via historical price as compared to current price or discounted current price; and store information, such as distance, store hours, and product availability. Allowing those people that choose to shop more sporadically, buying less items but more frequently through a period of time, as opposed to someone that bulk buys their shopping, to possibly change the shops they go to because of the application.
